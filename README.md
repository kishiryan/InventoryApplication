# InventoryApplication
Android Inventory Management Application

Briefly summarize the requirements and goals of the app you developed. What user needs was this app designed to address?

  This Android application was developed as a tool for users (free, individual, small business) to use a customizeable inventory management system to meet their requirements. Users can keep an active inventory while being able to check unique item ids, names, and item counts. Furthermore, the application is designed to notify the user via SMS when counts become low. 

What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful?

  The UI supports a login drop page which directs the user to the main application. The main application utilizes a multiscreen fragment based UI. The fragment pages consist of database display, add item, edit item, SMS messaging, and profile settings. These pages allow the core functionality to manage the inventory while also permitting changes to that experience via user settings. 

How did you approach the process of coding your app? What techniques or strategies did you use? How could those techniques or strategies be applied in the future?

  While approaching the development of my application, I focused on transforming user required features into functional code by keeping the user needs at the center of my workflow. I spent the majority of the alloted time developing these core features. I only worked on improvements once the core functionality was functional and stable. This technique was really effective for making sure the app functions correctly within the short development cycle. 

How did you test to ensure your code was functional? Why is this process important, and what did it reveal?

  Writing this code line by line allowed me to test constantly as I developed the code, but it may have been a bit inefficient. I think in the future it would be worthwhile to spend time developing testing tools to automate some of the functionality testing. That way I can allocate more time to developing futher functionality and features, while spending less on testing.

Consider the full app design and development process from initial planning to finalization. Where did you have to innovate to overcome a challenge?

  The application started at setup, deciding what apk and language to use. The entire UI was planned, designed, and created. Class file code was then created to add functionality to the UI system. Extensive additional testing was done to ensure that all code functionality was stable. Anything not within this critera was patched to get the application to a stable and functional application. This includes adding extra features, bug fixes, and removing unecessary or redundant code. 
  The most difficult part for me was getting settled into the project. I have never worked in Android Studio (I really like the system now), so that was my biggest challange. It took me a while to adjust to the unique quirks of the IDE as well as the new code calls/functions specific to Android (via Java). Once I settled into the changes it felt pretty smooth sailing from their.

While features such as that are becoming commonplace in applications, such features divide great apps from mediocre ones.
